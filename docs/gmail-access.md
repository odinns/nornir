# Gmail Access

Nornir imports Gmail through the official Gmail API with the read-only scope.

It does not scrape Gmail. It does not need your Google password. It stores an OAuth token beside the credentials file you provide, so keep that directory private.

Official Google docs:

- Google Takeout: https://support.google.com/accounts/answer/3024190
- Gmail API scopes: https://developers.google.com/workspace/gmail/api/auth/scopes
- Google Cloud OAuth clients: https://developers.google.com/identity/protocols/oauth2

## What Nornir Needs

- A Google Cloud project.
- Gmail API enabled for that project.
- An OAuth client credential JSON file.
- A local OAuth token generated by `php artisan gmail:auth`.
- A bounded Gmail query for every import.

The importer uses `https://www.googleapis.com/auth/gmail.readonly`.

## Create OAuth Credentials

1. Open Google Cloud Console.
2. Create or select a project.
3. Enable **Gmail API**.
4. Configure the OAuth consent screen.
5. Create an OAuth client.
6. Use a desktop app or installed-app style client when available.
7. Download the client JSON.
8. Save it somewhere ignored, for example:

```text
data/sources/gmail/credentials.json
```

If Google marks the app as testing, add your own Google account as a test user.

## Authorize Nornir

Run:

```bash
php artisan gmail:auth data/sources/gmail/credentials.json
```

Nornir prints a Google authorization URL. Open it, approve read-only Gmail access, then copy the returned `code` value back into the terminal.

The command writes:

```text
data/sources/gmail/token.json
```

Do not commit `credentials.json` or `token.json`.

## Import A Bounded Slice

Every Gmail import requires `--query`.

Examples:

```bash
php artisan import:gmail data/sources/gmail/credentials.json --query="from:someone@example.com OR to:someone@example.com"
php artisan import:gmail data/sources/gmail/credentials.json --query="after:2014/01/01 before:2015/01/01"
php artisan import:gmail data/sources/gmail/credentials.json --query="subject:(invoice OR receipt) after:2010/01/01"
```

Useful Gmail search operators:

- `from:`
- `to:`
- `cc:`
- `subject:`
- `after:YYYY/MM/DD`
- `before:YYYY/MM/DD`
- `has:attachment`
- `label:`
- quoted phrases

Use narrow queries first. "Import my whole mailbox" is how you turn a research tool into a swamp.

## Validate Or Dry-Run

```bash
php artisan import:gmail data/sources/gmail/credentials.json --query="from:someone@example.com" --validate-only
php artisan import:gmail data/sources/gmail/credentials.json --query="from:someone@example.com" --dry-run
```

## Backfill Plain Text

The importer preserves raw-ish message body material and normalized body text where available. If HTML plaintext extraction needs to be rerun:

```bash
php artisan gmail:backfill-body-plain
```

## Triage Important Mail

Nornir has a local important-mail triage command built for quick biography/reply review:

```bash
NORNIR_GMAIL_CREDENTIALS=data/sources/gmail/credentials.json php artisan gmail:triage-important --window="last 14 days"
```

Optional configuration lives in:

```text
config/gmail_triage.php
NORNIR_GMAIL_CREDENTIALS=/ignored/path/to/credentials.json
NORNIR_GMAIL_TRIAGE_RULES=/ignored/path/to/rules.json
```

Keep personal triage rules under `data/` or another ignored path. Do not commit real correspondence rules unless they are fully synthetic.

## Google Takeout

Google Takeout can export Gmail as MBOX:

```text
https://support.google.com/accounts/answer/3024190
```

Keep a Takeout archive for preservation if you care about long-term custody. Nornir's current implemented Gmail importer is API-first, so Takeout MBOX import is future work unless a dedicated importer is added.

## Revoking Access

You can revoke Nornir's OAuth access in your Google Account security settings.

Locally, delete:

```text
data/sources/gmail/token.json
```

Run `gmail:auth` again if you need a fresh token.
